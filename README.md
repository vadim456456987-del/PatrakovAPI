~~~
CustomersAPI - это пустой шаблон тестового проекта WebAPI на языке C#.
----------------------------------------------------------------------
Чтобы скачать проект, можно воспользоваться командой: 
   git clone https://github.com/CrHunta/CustomersAPI.git.

После этого необходимо выполнить следующие действия:
1. Перейти в папку с проектом и запустить файл restore.cmd.
2. Для нормальной работы программы необходима база данных PostgreSQL 
   версией не ниже 16. PostgreSQL должен быть установлен заранее. 
   В файле Scripts\ReadMe.txt дается краткая инструкция как восстановить 
   базу данных из логической резервной копии (файл Scripts\mycust_0.sql).
   Логин для подключения к базе данных должен быть postgres и пароль - 
   тоже postgres.
3. После того, как база данных создана, можно открыть файл CustomersApi.sln 
   в MS Visual Studio. Перед тем, как приступить к выполнению задания, решение 
   и проект необходимо переименовать (Название - фамилия студента, выполняющего 
   работу, в английской транскрипции).
4. Структура проекта:
..
├── appsettings.Development.json
├── appsettings.json
├── clean.cmd
├── Controllers
│   ├── AuthController.cs
│   └── CustomerController.cs
├── <Фамилия студента в английской транскрипции>.csproj
├── CustomersApi.http
├── <Фамилия студента в английской транскрипции>.sln
├── Data
│   └── AppDbContext.cs
├── Models
│   ├── AppUser.cs
│   ├── Auth
│   │   ├── AuthResponse.cs
│   │   ├── LoginRequest.cs
│   │   └── RegisterRequest.cs
│   ├── Customer.cs
│   └── JwtSettings.cs
├── Program.cs
├── Properties
│   └── launchSettings.json
├── publish.cmd
├── README.md
├── restore.cmd
├── Scripts
│   ├── mycust_0.sql
│   └── ReadMe.txt
├── Services
│   └── JwtTokenService.cs
├── start.cmd
└── wwwroot
    └── index.html

appsettings.json -   файл с настройками приложения. (Здесь, помимо прочего должна быть информация, 
                     необходимая для подключения к базе данных).
clean.cmd -          командный файл для очистки проекта от временных объектов.
Controllers -        папка, в которой хранятся классы контроллеров: 
                     AuthController.cs - контроллер аутентификации, 
                     CustomerController.cs - контроллер для доступа к таблице в базе данных.
Data -               папка, в которой находится класс AppDbContext.cs, описывающий контекст для работы с БД.
Models -             папка, в которой хранятся классы, описывающие модели данных.
 AppUser.cs -              класс для работы с таблицей, в которой хранятся данные, необходимые для 
                           аутентификации пользователей.
 Customer.cs -             класс, описывающий таблицу, данные котрой будут доступны через WebAPI.
 JwtSettings.cs -          класс, описывающий настройки JWT токена.
 Auth\AuthResponse.cs -    класс, описывающий объект, предаваемый в качестве ответа при аутентификации.
 Auth\LoginRequest.cs -    класс, описывающий объект, передаваемый в запросе на подключение.
 Auth\RegisterRequest.cs - класс, описывающий объект, передаваемый в запросе на регистрацию 
                           нового пользователя.
Program.cs -         основной файл программы.
Services\JwtTokenService.cs - файл, содержащий описания интерфейса и сервиса для генерации JWT-токенов, 
                              являющихся основой механизма аутентификации пользователей.
wwwroot\index.html - статическая WEB-страница, которая при помощи функций, написанных на JavaScript-e, 
                     обращается к основному приложению, используя вызовы WebAPI (применяется протокол HTTP и 
                     формат данных JSON). При помощи этой страницы необходимо реализовать основные 
                     CRUD-операции с таблицей, которая описана моделью Models\Customer.cs.

  Программа представляет из себя сервис WebAPI (данные передаются клиенту по протоколу HTTP в формате JSON), 
который позволяет выполнять CRUD-операции над таблицей в базе данных.
Задача: написать реализацию всех необходимых для функционирования программы классов, а также реализовать 
взаимодействие с программой-сервисом WebAPI через Web-страницу index.html, которая в данном случае играет роль клиента 
и "общается" с нашим сервисом посредством вызова функций, написанных на JavaScript. 
Страницу index.html допускается сгенерировать при помощи ИИ.

5. В общем виде процесс использования готовой программы можно описать так: 
   - Используя браузер, заходим на ресурс, например, http://localhost:5000;
   - Регистрируемся как новый пользователь, проходим аутентификацию;
   - Попадаем на страницу http://localhost:5000/index.html на которой отображается 
     содержимое таблицы из базы данных, а также через эту же страницу имеется возможность 
     добавить новую строку в таблицу, изменить выбранную строку таблицы, удалить выбранную 
     строку из таблицы. 

6. После того, как программа написана и протестирована, необходимо выложить проект на GitHub и 
   прислать мне ссылку на репозиторий по электронной почте.
~~~
